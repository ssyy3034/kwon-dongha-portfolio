# 그래프 표현 방식 정리

## 1) **인접 리스트 (Adjacency List)**

- **구조**
    
    - 각 정점 $u$에 대해 연결된 정점들을 **리스트**로 저장
        
    - 구현: 배열(혹은 해시테이블) + 연결 리스트(혹은 동적 배열)
        
    - 예시:
        
        ```
        1: [2, 5]
        2: [1, 3, 5]
        3: [2, 4]
        4: [3, 5]
        5: [1, 2, 4]
        ```
        
- **공간 복잡도**: $O(V+E)$  
    → 각 정점 헤더 $O(V)$ + 간선 저장 $O(E)$
    
- **시간 복잡도**
    
    - 간선 $(u,v)$ 존재 여부 확인: $O(\deg(u))$ (최악 $O(V)$)
        
    - 인접 정점 나열: $O(\deg(u))$
        
- **장점**
    
    - **희소 그래프(sparse)** 에서 공간 절약 ($E \ll V^2$)
        
    - DFS, BFS 구현에 자연스럽게 사용 가능 → 전체 탐색 $O(V+E)$
        
- **단점**
    
    - 간선 존재 여부를 빠르게 확인하기 어렵다
        

---

## 2) **인접 행렬 (Adjacency Matrix)**

- **구조**
    
    - $V \times V$ 크기의 2차원 배열 $A$
        
    - $A[u][v] = 1$ (또는 가중치 $w$)이면 간선 존재, 아니면 0
        
    - 예시:
        
        ```
           1  2  3  4  5
        1 [0, 1, 0, 0, 1]
        2 [1, 0, 1, 0, 1]
        3 [0, 1, 0, 1, 0]
        4 [0, 0, 1, 0, 1]
        5 [1, 1, 0, 1, 0]
        ```
        
- **공간 복잡도**: $O(V^2)$  
    → 희소 그래프에서는 메모리 낭비 심함
    
- **시간 복잡도**
    
    - 간선 $(u,v)$ 존재 여부 확인: $O(1)$
        
    - 인접 정점 나열: $O(V)$
        
- **장점**
    
    - 구현 간단
        
    - **밀집 그래프(dense)**에서 효율적
        
    - 간선 존재 여부 확인이 매우 빠름
        
- **단점**
    
    - 희소 그래프에서 공간 낭비가 크다
        

---

## 3) **비교 요약**

|구분|인접 리스트|인접 행렬|
|---|---|---|
|**공간 복잡도**|$O(V+E)$|$O(V^2)$|
|**간선 존재 확인**|$O(\deg(u))$ (최악 $O(V)$)|$O(1)$|
|**인접 정점 순회**|$O(\deg(u))$|$O(V)$|
|**희소 그래프 (sparse)**|메모리 효율적 👍|비효율적 👎|
|**밀집 그래프 (dense)**|비효율적 👎|효율적 👍|
|**알고리즘 사용 예**|BFS, DFS, Dijkstra (보통 리스트 기반)|플로이드-워셜(모든 쌍 최단 경로) 등|

- **희소 그래프 → 인접 리스트**
    
- **밀집 그래프 → 인접 행렬**
