## 2. C 기초

### 2.1 컴파일과 링크 과정

C 프로그램은 세 단계를 거쳐 실행 파일이 된다:

1. **전처리 (Preprocessing)**
    
    - `#`으로 시작하는 지시자를 처리
        
    - 예: `#include`, `#define`
        
2. **컴파일 (Compilation)**
    
    - 전처리된 코드를 컴파일러가 기계어로 번역
        
    - 목적 파일(`.o`) 생성
        
3. **링킹 (Linking)**
    
    - 링커가 목적 코드와 라이브러리 코드를 합쳐 실행 파일 생성
        

---

### 2.2 C 프로그램 구성 요소

- **지시자**: `#`으로 시작, 세미콜론 불필요
    
- **함수**: `main()` 필수, `return 0;` 권장 (정상 종료)
    
- **구문**: 실행 시 실행되는 명령들, 반드시 세미콜론으로 끝남
    

---

### 2.3 printf / scanf

- `printf`는 출력 → 값만 필요, `&` 필요 없음
    
- `scanf`는 입력 → 변수에 값을 저장해야 하므로 **주소(`&`) 필요**
    

---

### 2.4 기타 기본 규칙

- `printf`는 줄바꿈하려면 `\n` 필요
    
- `switch` 문은 각 `case` 뒤에 반드시 `break`
    
- 주석은 전처리 시 스페이스로 대체됨
    
- 표현식의 끝에서는 모든 증감 연산이 적용된 상태여야 함

## 3. GCC (GNU Compiler Collection)

- **GNU**: 유료였던 UNIX 소프트웨어에 대항해 무료로 다시 만든 프로젝트
    
- **GCC 특징**
    
    - 무료, 여러 언어 지원
        
    - CPU에 맞는 기계어 생성
        
    - UNIX 계열 OS에서 널리 사용
        
    - 상용 목적 사용 가능

## 4. unsigned 언제 쓰나?

- **크기·개수·인덱스** (음수 없음)  
    → `size_t`, `uint32_t`
    
- **비트마스크/플래그**  
    → `uint32_t`, `uint64_t`
    
- **모듈러/래핑 카운터**  
    → `uint32_t`
    
- **하드웨어/디스크 섹터 번호**  
    → `block_sector_t`
    

> 🔎 참고:
> 
> - `unsigned` 오버플로우는 **모듈러 2^N**으로 정의됨
>     
> - `signed` 오버플로우는 **UB(정의되지 않음)**
>

## 5. 배열 인덱스 주의점

- C는 배열 범위 검사를 강제하지 않는다
    
- 첨자가 범위를 벗어나면 → 프로그램 동작 불능 (UB)
    

### 흔한 실수

```c
int a[10], i;

for (i = 1; i <= 10; ++i)   // 잘못된 루프
    a[i] = 0;
```

- 배열 `a[10]`은 존재하지 않음 (`0 ~ 9`까지만 있음)
    
- `a[10]`에 쓰면 메모리 침범 발생
    
- 만약 `i`가 그 메모리 위치에 있었다면 값이 바뀌어 무한 루프 가능
