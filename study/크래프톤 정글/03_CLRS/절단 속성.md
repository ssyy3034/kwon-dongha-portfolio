# ✂️ 절단 속성 (Cut Property) 정리

## 1) 절단(Cut) 정의

- 그래프 $G=(V,E)$의 정점 집합 $V$를 두 부분 $S$와 $V\setminus S$로 나누는 것
    
- 이때 $S$와 $V\setminus S$를 잇는 간선 집합을 **cut-set**이라 부른다
    

---

## 2) 절단 속성 (Cut Property)

> **어떤 절단 $(S, V\setminus S)$을 가로지르는 간선들 중 최소 가중치 간선(light edge)은 반드시 어떤 MST에 포함된다.**

### ✨ 직관

- MST는 "싸고 안전한 연결"만 포함해야 한다
    
- 절단을 만들면, $S$와 $V\setminus S$를 연결해야 그래프가 하나로 이어진다
    
- 그 중 제일 가벼운 간선을 쓰는 게 항상 이득
    
- 만약 MST에 그 간선이 없다면? → 더 무거운 간선이 들어가 있을 텐데, **교환 논증(exchange argument)** 으로 바꿔치기 가능 → 항상 light edge를 넣을 수 있음
    

---

## 3) 두 알고리즘에서의 적용

### 🟦 Kruskal 알고리즘

1. 간선을 가중치 오름차순으로 정렬
    
2. 작은 간선을 보면서 **서로 다른 집합을 잇는지 확인**
    
3. 두 정점이 다른 집합이면, 그 간선은 어떤 절단을 가로지르는 **최소 간선(light edge)**임  
    → 절단 속성에 따라 MST에 안전하게 포함
    
4. 같은 집합이면 이미 연결돼 있으므로 사이클 → 제외
    

👉 Kruskal은 “모든 절단을 동시에 고려하면서, 가장 싼 간선을 고르면 안전하다”는 절단 속성을 직접 활용하는 셈

---

### 🟩 Prim 알고리즘

1. 이미 선택한 정점 집합 = $S$, 아직 안 뽑은 집합 = $V\setminus S$
    
2. 절단 $S \mid V\setminus S$를 만든 셈
    
3. 이 절단을 가로지르는 간선 중 **최소 간선(light edge)**을 선택  
    → 절단 속성에 의해 항상 MST에 포함됨
    
4. 선택된 간선이 연결하는 정점을 $S$에 합류시킴
    

👉 Prim은 “하나의 절단을 계속 키워나가면서 light edge만 고른다”는 구조

---

## 4) 간단 예시

그래프 (간선 가중치):

```
A --1-- B
|      /
3    2
|  /
C
```

간선: AB=1, BC=2, AC=3

- **절단 {A} | {B,C}**
    
    - cut-set = {AB(1), AC(3)}
        
    - light edge = AB(1) → MST에 포함
        
- **Kruskal 진행**
    
    - 정렬: AB(1), BC(2), AC(3)
        
    - AB(1) 선택 (절단 {A}|{B,C}의 최소 간선)
        
    - BC(2) 선택 (절단 {A,B}|{C}의 최소 간선)
        
    - MST = {AB, BC}
        
- **Prim 진행 (A 시작)**
    
    - S={A}, 절단 간선 = {AB(1), AC(3)} → AB 선택
        
    - S={A,B}, 절단 간선 = {BC(2), AC(3)} → BC 선택
        
    - MST = {AB, BC}
        

👉 두 방식 모두, **각 단계에서 light edge를 골랐기 때문에** 결과가 MST로 수렴

## ✅ 한 줄 요약

- 절단 속성: “절단을 넘는 간선 중 가장 가벼운 간선은 MST에 반드시 포함된다”
    
- Kruskal: 모든 간선을 작은 것부터 보며 **사이클 없는 절단의 light edge만 채택**
    
- Prim: 현재 트리와 외부 사이의 **절단 light edge만 선택**
