# 인터넷 연결 (Internet Connection)

## 1. 개요

인터넷 연결은 **전 세계의 호스트(컴퓨터)** 들이  
**TCP/IP 프로토콜 스택**을 통해 서로 데이터를 주고받는 구조를 말한다.  
이는 [[LAN]]·[[WAN]] 등 **서로 다른 네트워크 기술**을  
**라우터(router)** 로 이어 붙여 하나의 거대한 **논리적 네트워크**로 만든 것이다.

---

## 2. 핵심 구성요소

### (1) 호스트 (Host)

인터넷에 연결된 모든 컴퓨터를 뜻하며,  
각 호스트는 고유한 [[IP]] 주소를 가진다.  
모든 인터넷 애플리케이션(웹, 이메일, SSH 등)은  
이 주소를 기반으로 통신한다.

### (2) 프로토콜 (Protocol)

인터넷의 통신 규칙은 **TCP/IP 프로토콜 스택**에 의해 정의된다.

- **IP (Internet Protocol)** : 데이터그램을 목적지 호스트로 전달하는 역할
    
- **UDP (User Datagram Protocol)** : 프로세스 간 빠른 비연결형 통신
    
- **TCP (Transmission Control Protocol)** : 신뢰성 있는 양방향 연결  
    이들은 함께 동작하며, 네트워크 계층부터 애플리케이션 계층까지의  
    데이터 전송을 담당한다.
    

### (3) 소켓 (Socket)

소켓은 연결의 끝점이다.  
각 소켓은 **IP 주소 + 포트 번호**로 구성된 **소켓 주소(socket address)** 를 가진다.  
예를 들어,

```
클라이언트 : 128.2.194.242:51213  
서버 : 208.216.181.15:80
```

이 두 소켓 주소의 쌍(`(cliaddr:cliport, servaddr:servport)`)이  
하나의 인터넷 연결을 **유일하게 식별**한다.

---

## 3. 연결 동작 과정

1. **클라이언트 요청(Client Request)**
    
    - 클라이언트 프로세스가 `connect()`를 호출하면,  
        커널은 임시 포트(ephemeral port)를 자동 할당하고  
        서버의 고정 포트(예: 80번, 25번 등)로 요청을 전송한다.
        
2. **라우팅 (Routing)**
    
    - 패킷은 여러 **라우터(router)** 를 거치며 최적 경로로 전달된다.  
        라우터는 IP 헤더를 보고 다음 목적지(다음 홉)를 결정한다.
        
3. **서버 수신(Server Accept)**
    
    - 서버는 해당 포트를 리스닝 중이며(`listen()`),  
        연결 요청이 오면 커널이 소켓을 새로 생성(`accept()`)하여  
        클라이언트와의 TCP 연결을 수립한다.
        
4. **데이터 송수신 (Data Transmission)**
    
    - 이후에는 TCP의 신뢰성 메커니즘에 따라 데이터 스트림이 교환된다.  
        IP 계층에서는 각각의 세그먼트가 **데이터그램(datagram)** 으로 전달된다.
        
5. **연결 종료 (Connection Close)**
    
    - 통신이 끝나면 `close()` 호출로 양쪽에서 FIN → ACK 절차를 거쳐 종료된다.
        

---

## 4. 주소 체계

### (1) IP 주소

- **IPv4** : 32비트 주소 (예: `192.168.0.1`)
    
- **IPv6** : 128비트 주소 (예: `2001:0db8::1`)  
    IPv6는 주소 부족 문제를 해결하기 위해 제안되었지만,  
    여전히 인터넷 트래픽의 대부분은 IPv4 기반이다.
    

### (2) 도메인 이름

- 사람이 읽기 쉬운 문자열로 IP 주소를 식별하기 위한 체계
    
- 예: `www.example.com` → `93.184.216.34`
    
- 변환은 [[DNS]](Domain Name System)이 담당한다.
    

---

## 5. 인터넷 연결의 특징

|항목|설명|
|---|---|
|**비동기 전송**|서로 다른 네트워크 환경에서도 독립적으로 데이터 전송 가능|
|**캡슐화(Encapsulation)**|LAN 프레임 안에 IP 패킷, 그 안에 TCP/UDP 세그먼트가 포함됨|
|**Best-Effort 서비스**|IP는 손실 복구나 순서 보장을 하지 않음|
|**종단 간 신뢰성**|TCP 계층이 오류 감지·재전송으로 보완|
|**라우팅 기반 구조**|각 데이터그램은 서로 다른 경로를 통해 목적지로 이동 가능|

---

## 6. 요약

> 인터넷 연결은 **전 세계 호스트들이 TCP/IP 스택을 통해 논리적으로 하나로 묶인 구조**이다.  
> 각 연결은 **소켓 주소 쌍**으로 식별되며, **IP가 데이터 전송을, TCP/UDP가 신뢰성과 속도를** 담당한다.  
> 이 구조 덕분에 서로 다른 기술의 네트워크가 **하나의 글로벌 인터넷**으로 동작할 수 있다.
