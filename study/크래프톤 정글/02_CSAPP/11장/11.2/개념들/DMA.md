**DMA(Direct Memory Access)**는 **CPU의 개입 없이**, I/O 장치가 메인 메모리에 직접 접근하여 데이터를 읽거나 쓸 수 있도록 하는 컴퓨터 시스템의 기능입니다.

CPU가 모든 데이터 전송을 중재해야 하는 기존의 방식(Programmed I/O)과 달리, DMA는 데이터 복사 작업을 **DMA 컨트롤러(DMAC)**라는 특수한 하드웨어에 위임하여 CPU가 다른 중요한 연산을 계속 수행할 수 있도록 합니다.

---

### DMA의 목적

DMA의 주된 목적은 대용량 데이터 전송 시 발생하는 **CPU의 부하를 줄여 시스템 전체의 성능을 향상**시키는 것입니다.

- **CPU의 역할 최소화**: 디스크나 [[네트워크 어댑터]] 같은 I/O 장치와 메모리 간의 데이터 복사는 CPU에게 단순 반복적인 작업입니다. DMA는 이 작업을 CPU로부터 오프로드(offload)하여, CPU가 더 복잡한 연산에 집중할 수 있도록 합니다.
    
- **고속 데이터 전송**: DMA 컨트롤러는 데이터 복사 작업에만 특화되어 있어, CPU가 중간에 개입하는 것보다 더 빠른 속도로 데이터를 전송할 수 있습니다.
    

---

### DMA의 동작 과정

1. **CPU의 초기 설정**:
    
    - CPU는 데이터 전송에 필요한 정보들을 DMA 컨트롤러의 레지스터에 설정합니다.
        
        - **전송 방향**: I/O 장치 → 메모리 (읽기) 또는 메모리 → I/O 장치 (쓰기)
            
        - **소스(Source) 주소**: 데이터가 시작되는 메모리 또는 I/O 장치의 주소
            
        - **목적지(Destination) 주소**: 데이터가 기록될 메모리 또는 I/O 장치의 주소
            
        - **데이터 크기**: 전송할 데이터의 총량
            
2. **DMA 컨트롤러의 작업 수행**:
    
    - CPU로부터 작업 지시를 받은 DMA 컨트롤러는 시스템 버스의 제어권을 획득합니다.
        
    - DMA 컨트롤러는 설정된 정보에 따라, CPU의 도움 없이 소스에서 목적지로 직접 데이터를 복사하기 시작합니다. CS:APP 11장에서는 네트워크 어댑터가 네트워크로부터 받은 데이터를 메모리로 복사할 때 이 DMA 전송 방식을 사용한다고 설명합니다.
        
3. **작업 완료 및 인터럽트**:
    
    - 데이터 전송이 모두 완료되면, DMA 컨트롤러는 CPU에 **인터럽트(Interrupt)** 신호를 보냅니다.
        
    - 이 신호를 받은 CPU는 데이터 전송이 완료되었음을 인지하고 다음 작업을 수행합니다.
        

이처럼 DMA는 CPU가 데이터 전송의 시작과 끝만 관여하고, 실제 데이터 복사 과정에서는 다른 일을 할 수 있도록 만들어주는 매우 효율적인 I/O 방식입니다.
