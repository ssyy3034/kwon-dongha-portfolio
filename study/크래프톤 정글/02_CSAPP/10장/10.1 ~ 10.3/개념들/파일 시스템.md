##  파일 시스템(File System)이란 무엇인가?

파일 시스템은 운영체제가 저장 장치(하드 디스크, SSD, USB 등)에 있는 **파일을 어떻게 저장하고, 구성하고, 찾고, 관리할지에 대한 규칙과 구조**를 정의한 것입니다.
### 파일 시스템의 핵심 역할

- **데이터 저장 및 관리:** 파일을 물리적인 디스크 공간에 조각내어 저장하고, 이 조각들이 어디에 있는지 추적합니다.
    
- **계층적 구조 제공:** 사용자가 파일을 쉽게 찾고 정리할 수 있도록 **디렉터리(폴더)와 하위 디렉터리** 구조를 제공합니다.
    
- **메타데이터 관리:** 파일의 실제 내용 외에 부가 정보, 즉 **메타데이터**를 관리합니다.
    
    - 파일 이름, 크기, 생성 날짜, 수정 날짜
        
    - 소유자 및 그룹 정보, **접근 권한(읽기/쓰기/실행)**
        
- **보안 및 접근 제어:** 어떤 사용자가 어떤 파일에 접근할 수 있는지 제어하는 보안 기능을 제공합니다.
    
- **안정성 보장:** **저널링(Journaling)** 과 같은 기술을 통해 예기치 않은 시스템 종료로부터 파일 시스템의 일관성을 보호하고 데이터 손상을 방지합니다.

## ## 주요 파일 시스템 종류 비교

각 파일 시스템은 서로 다른 특징과 장단점을 가지며, 특정 운영체제나 사용 목적에 맞게 발전해 왔습니다.

| **특징 구분**  | **FAT32 (File Allocation Table 32)**           | **NTFS (New Technology File System)**               | **ext4 (Fourth Extended Filesystem)**           | **APFS (Apple File System)**                             |
| ---------- | ---------------------------------------------- | --------------------------------------------------- | ----------------------------------------------- | -------------------------------------------------------- |
| **주요 OS**  | 윈도우, macOS, 리눅스 (범용)                           | 윈도우 (Windows)                                       | 리눅스 (Linux)                                     | 애플 macOS, iOS                                            |
| **핵심 구조**  | **파일 할당 테이블(FAT)**이라는 간단한 목록으로 파일 조각들을 연결합니다.  | **마스터 파일 테이블(MFT)**이라는 강력한 데이터베이스로 모든 파일 정보를 관리합니다. | **Inode**를 사용하여 파일 메타데이터를 관리하고 저널링으로 안정성을 높입니다. | SSD에 최적화된 **Copy-on-Write** 방식을 사용해 데이터 안정성과 속도를 극대화합니다. |
| **보안**     | ❌ **접근 제어 기능 없음**                              | ✅ **파일/폴더별 접근 제어(ACL)**를 지원하여 보안성이 뛰어납니다.           | ✅ **전통적인 유닉스 권한** (소유자/그룹/기타)을 지원하여 보안성이 좋습니다.  | ✅ **파일별 암호화** 등 강력한 보안 기능을 기본으로 제공합니다.                   |
| **안정성**    | ❌ **저널링 기능 없음** (전원 차단 시 데이터 손상 위험 큼)          | ✅ **저널링 지원**으로 시스템 충돌 시 파일 시스템을 빠르게 복구할 수 있습니다.     | ✅ **저널링 지원**으로 안정성이 매우 높습니다.                    | ✅ **Copy-on-Write, 스냅샷** 등 데이터 무결성을 보장하는 고급 기능을 지원합니다.   |
| **최대 크기**  | 파일: **4GB** 볼륨: **32GB** (윈도우 포맷 기준)           | 파일/볼륨: **사실상 거의 무제한** (페타바이트 이상)                    | 파일/볼륨: **사실상 거의 무제한** (엑사바이트 이상)                | 파일/볼륨: **사실상 거의 무제한**                                    |
| **주요 사용처** | **호환성**이 가장 중요할 때 (USB 메모리, SD 카드 등 이동식 저장 장치) | **윈도우 운영체제 설치 드라이브**, 대용량 내장/외장 하드 디스크              | **대부분의 리눅스 배포판**의 기본 파일 시스템                     | **애플 기기(Mac, iPhone)의 내장 SSD**에 최적화되어 사용됩니다.             |
| **도서관 비유** | **동네 작은 도서관** (간단한 카드 목록, 누구나 출입 가능)           | **현대적인 대학 도서관** (중앙 컴퓨터 시스템, 출입증 기반 보안)             | **체계적인 국립 도서관** (정교한 분류 체계, 안정적인 운영)            | **미래형 스마트 도서관** (빠른 검색, 데이터 자동 백업)                       |
