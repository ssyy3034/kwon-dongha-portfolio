# 🌐 HTTP vs HTTPS 차이 (정리 중심)

|항목|HTTP|HTTPS|
|---|---|---|
|**약자**|HyperText Transfer Protocol|HyperText Transfer Protocol Secure|
|**포트 번호**|80|443|
|**보안성**|없음 (평문 전송)|있음 (암호화된 전송)|
|**암호화**|❌|✅ (TLS 사용)|
|**신뢰성**|도청·위조 가능|전송 중 도청·위조 방지|
|**인증서 필요**|❌|✅ 서버 인증서 필요 (CA 발급)|
|**브라우저 표시**|🔓 주소창에 http://|🔒 주소창에 🔒 또는 https://|
|**사용 예시**|사내 테스트 서버, 비공개 API|로그인, 결제, 사용자 정보 전송 등|

> ✅ HTTPS = HTTP + TLS (Transport Layer Security)

# 🔐 SSL/TLS란?


## ░ 정의

**SSL/TLS는 전송 계층 보안 프로토콜**  
→ 애플리케이션 계층(예: HTTP)의 데이터를 **암호화·인증·무결성 보호**하는 보안 프로토콜

- SSL: 이전 명칭 (v3.0 이후 폐기됨)
    
- TLS: 현재 표준 (버전 1.2 / 1.3 사용 중)
    
- HTTPS = TLS 위에서 HTTP가 동작하는 구조 (`HTTP over TLS`)
    

---

## ░ TLS의 3대 핵심 목표

|목표|설명|
|---|---|
|**기밀성 (Confidentiality)**|데이터를 암호화하여 도청 방지 (대칭키 사용)|
|**무결성 (Integrity)**|데이터 변조 여부 검증 (HMAC 또는 AEAD)|
|**인증 (Authentication)**|상대(서버)가 진짜인지 확인 (X.509 인증서 + CA 신뢰 체계)|

> ✅ 최신 TLS 1.3은 모든 키 교환을 **ECDHE 기반**으로 하여 **PFS (Perfect Forward Secrecy)** 보장

---

## ░ TLS 1.3 구조 요약

```plaintext
Client                               Server
 ──────────────────────────────────────────────────────
 ClientHello  ─────────────→    (지원 암호군, 키 공유)
                          ←──────────── ServerHello
                            + 인증서 + 핸드셰이크 완료
 Finished     ─────────────→    (양방향 키 생성 완료)
```

- 🔐 이후 모든 데이터는 **세션 키로 암호화됨**
    
- 💡 키 교환: **ECDHE** 방식 → 서버/클라이언트 비밀키로 세션 키 파생
    
- 🔁 세션 재개: TLS 세션 티켓 또는 세션 ID
    

---

## ░ 주요 개념 용어

|용어|설명|
|---|---|
|**X.509 인증서**|서버 공개키와 도메인 포함된 디지털 문서. CA가 서명|
|**CA (인증기관)**|인증서를 발급하고 브라우저는 이를 신뢰|
|**SNI (Server Name Indication)**|같은 서버에 여러 도메인이 있을 경우 구분|
|**ALPN**|TLS 중 HTTP/1.1, HTTP/2, HTTP/3 중 선택|
|**AEAD**|AES-GCM, ChaCha20 등 암호화+무결성 통합 알고리즘|

# 💡 HTTP와 HTTPS 통신 흐름 비교

### 📡 HTTP (평문)

```plaintext
[브라우저] ── 평문 요청 ──▶ [서버]
          ◀─ 평문 응답 ───
```

- 암호화 없음, 누구나 중간에서 **내용 열람 가능**
    
- 쿠키, 세션, 비밀번호, API 키 등 유출 위험
    

---

### 🔒 HTTPS (TLS 포함)

```plaintext
[브라우저] ── TLS 핸드셰이크 (키 공유 + 인증) ──▶ [서버]
                ▼
         이후 모든 데이터는 암호화됨
```

- 🔒 전송 경로에서 **도청, 변조, 위조 불가능**
    
- 📜 서버 인증서로 신뢰 확인
    
- 🧠 세션 키로 빠른 암호화 통신 가능

# 🧪 검색 기반 최신 정보 반영

|기준|현재 표준|
|---|---|
|**TLS 버전**|1.3 (RFC 8446), 1.2 백업용|
|**SSL 사용 여부**|**비권장** (SSLv3 및 TLS 1.0/1.1은 보안 취약)|
|**암호 알고리즘**|ECDHE 키교환 + AES-GCM / ChaCha20-Poly1305|
|**인증서**|Let’s Encrypt 무료 발급 + 자동 갱신 사용 증가|

# ✅ 정리 요약

- HTTP는 **암호화 없음** → 민감 정보 유출 위험
    
- HTTPS는 **TLS로 암호화된 HTTP**  
    → 기밀성, 무결성, 서버 인증 모두 보장
    
- TLS는 **SSL의 최신/안전 버전**
    
- TLS 1.3은 **빠르고 안전**하며 최신 웹 서비스의 기본
