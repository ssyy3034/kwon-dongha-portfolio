DI란: 객체가 자기가 사용할 의존 객체를 직접 생성하지 않고, 외부에서 주입받는 것.

왜 필요한가: 구체 클래스에 직접 의존하면 두 가지 문제가 생긴다. 첫째, 구현체를 교체할 때 클라이언트 코드를 수정해야 한다(OCP 위반). 둘째, 상위 모듈이 하위 모듈의 구체 구현을 알게 된다(DIP 위반). 인터페이스를 도입해도 객체 생성을 클라이언트가 직접 하면 여전히 구체 클래스를 아는 것이므로, 생성 책임까지 외부로 넘기는 DI가 필요하다.

역할과 구현의 분리
역할(인터페이스)과 구현(구체 클래스)을 분리하고, Config에서 어떤 구현체를 빈으로 등록할지 결정한다. 구현체를 교체할 때 Config만 수정하면 되고, 주입받는 쪽은 인터페이스 타입으로만 의존하므로 코드 변경이 없다.
