## ## 웹 서버 (Web Server)

**웹 서버**는 HTTP 요청을 받아 처리하고, 그 결과를 클라이언트(주로 웹 브라우저)에게 응답으로 보내주는 소프트웨어입니다. 웹사이트의 '문지기'이자 '안내원' 역할을 합니다.

웹 서버의 주된 임무는 두 가지입니다.

* **1. 정적 콘텐츠(Static Contents) 제공**:
    * **역할**: 요청이 들어오면, 미리 서버 컴퓨터에 저장된 파일(HTML, CSS, JavaScript, 이미지 등)을 **변경 없이 그대로** 클라이언트에게 전송합니다.
    * **예시**: 회사의 소개 페이지나 블로그의 이미지처럼, 누가 언제 접속하든 항상 동일한 내용을 보여주는 경우입니다.

* **2. 동적 콘텐츠(Dynamic Contents) 제공**:
    * **역할**: 요청이 들어오면, 단순 파일 전송이 아닌 **별도의 프로그램을 실행**하여 그 실행 결과를 클라이언트에게 전송합니다.
    * **예시**: 로그인, 검색 결과, 온라인 쇼핑몰의 장바구니처럼, 사용자의 요청이나 상황에 따라 내용이 실시간으로 바뀌는 경우입니다. 이때 웹 서버는 CGI와 같은 방식을 통해 외부 프로그램을 호출합니다.

**대표적인 웹 서버**: Apache, Nginx

## ## CGI (Common Gateway Interface)

**CGI**는 **웹 서버가 동적 콘텐츠를 생성하기 위해 외부 프로그램과 소통하는 방법에 대한 표준 규약(Interface)**입니다. CGI 자체는 프로그래밍 언어가 아니라, 웹 서버와 외부 프로그램 사이의 '통신 규칙'입니다.

#### ### CGI의 동작 과정
1.  클라이언트(웹 브라우저)가 동적 페이지(예: `login.py`)를 요청합니다.
2.  웹 서버는 이 요청을 받고, 이것이 단순 파일이 아니라 실행해야 할 프로그램임을 인지합니다.
3.  웹 서버는 **CGI 규약에 따라** `login.py` 프로그램을 **새로운 프로세스로 실행**시킵니다. 이때 요청에 담긴 정보(사용자 ID, 비밀번호 등)를 프로그램에 전달합니다.
4.  `login.py` 프로그램은 전달받은 정보로 데이터베이스를 조회하는 등의 작업을 수행한 후, 그 결과를 **HTML 문서 형태로 표준 출력(stdout)**에 씁니다.
5.  웹 서버는 실행된 프로그램의 표준 출력을 그대로 받아, HTTP 응답에 담아 클라이언트에게 전송합니다.
6.  클라이언트의 브라우저는 이 HTML을 받아 화면에 렌더링합니다.

CGI는 요청마다 새로운 프로세스를 생성하기 때문에 비효율적인 단점이 있어, 오늘날에는 이 단점을 개선한 FastCGI, WSGI(Python), PSGI(Perl) 같은 더 발전된 인터페이스를 많이 사용합니다.

## ## MIME 타입 (MIME Type)

**MIME 타입**은 웹 서버가 클라이언트에게 보내는 데이터가 **어떤 종류의 파일인지 알려주는 식별자**입니다. 브라우저는 이 정보를 보고 받은 데이터를 어떻게 처리할지 결정합니다.

* **형식**: `주 타입/부 타입` (예: `text/html`, `image/jpeg`)
* **역할**: 이 MIME 타입은 HTTP 응답 헤더의 **`Content-Type`** 필드에 담겨 전송됩니다.
* **비유**: 택배 상자에 붙어있는 '내용물' 스티커와 같습니다. '서류'라고 적혀 있으면 열어서 읽고, '유리 제품'이라고 적혀 있으면 조심스럽게 다루듯이, 브라우저는 MIME 타입을 보고 행동을 결정합니다.

#### ### 주요 MIME 타입 예시
* `text/html`: 브라우저는 이 데이터를 HTML 문서로 해석하여 렌더링합니다.
* `image/jpeg`: JPEG 이미지로 화면에 표시합니다.
* `application/json`: JSON 데이터로, 주로 API 통신에서 자바스크립트가 처리합니다.
* `application/pdf`: PDF 뷰어를 실행하거나 파일을 다운로드하도록 유도합니다.
* `text/plain`: 일반 텍스트 파일로, 보통 그대로 화면에 표시됩니다.
