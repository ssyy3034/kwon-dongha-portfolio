# 패킷 (Packet)

## 1. 정의

패킷은 **네트워크 계층(IP)** 이 서로 다른 네트워크 간 데이터를 전달하기 위해 정의한  
**표준화된 데이터 단위**이다.  
각 패킷은 독립적으로 라우팅되며, IP 계층에서는 이를 **[[데이터그램]](datagram)** 이라고 부른다.

IP는 패킷이 손실되거나 순서가 뒤바뀌어도 복구하지 않는 **비신뢰성(Best-Effort)** 전송 방식을 사용한다.

---

## 2. 구조

패킷은 크게 **헤더(Header)** 와 **페이로드(Payload)** 로 구성된다.

| 구성 요소    | 설명                                                        |
| -------- | --------------------------------------------------------- |
| **헤더**   | 패킷의 크기, 송신지/목적지의 [[IP 주소]], 프로토콜 식별자, TTL, 체크섬 등 제어 정보 포함 |
| **페이로드** | 상위 계층(예: [[TCP]], [[UDP]])에서 전달된 실제 데이터 세그먼트              |

---

## 3. 데이터그램과 신뢰성

- IP 계층은 패킷을 **데이터그램(datagram)** 형태로 전송하며,  
    각 데이터그램은 독립적으로 라우팅된다.
    
- 네트워크 혼잡, 라우팅 경로 변경 등으로
    데이터그램은 순서가 달라지거나 손실될 수 있다.
    
- 이러한 문제는 상위 계층(예: TCP)에서 재조립과 재전송으로 해결한다.
    

---

## 4. MTU (Maximum Transmission Unit)과 단편화

- IP 패킷은 데이터 링크 계층의 **MTU(최대 전송 단위)** 를 초과할 수 없다.
    
- 일반적인 **이더넷 MTU는 1500바이트**이다.
    
- 이를 초과하는 경우, IP 계층은 패킷을 여러 조각(fragment)으로 나누어 전송한다.
    
- 수신 측은 각 조각을 다시 조립(reassembly)하여 원래의 패킷을 복원한다.
    

---

## 5. 캡슐화 (Encapsulation)

패킷은 상위 계층의 데이터를 감싸는 형태로 구성된다.

애플리케이션 데이터
        ↓
전송 계층 (TCP / UDP)
        → 세그먼트 (Segment)
        ↓
네트워크 계층 (IP)
        → 패킷 (Packet)
        ↓
데이터 링크 계층 (Ethernet 등)
        → 프레임 (Frame)
        ↓
물리 계층
        → 비트 신호 (Bits)


이 과정을 **캡슐화(encapsulation)** 라고 하며,  
수신 측에서는 역순으로 해석(decapsulation)한다.

---

## 6. 요약

> 패킷은 **IP 계층의 데이터 단위**로,  
> 상위 계층의 세그먼트를 감싸고, 하위 계층의 프레임 속에 담겨 전송된다.  
> IP는 이를 데이터그램으로 처리하며,  
> 신뢰성 없는 대신 **유연하고 확장 가능한 전송 구조**를 제공한다.
