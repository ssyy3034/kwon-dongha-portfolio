# Aidiary - 마음으로 그리는 AI 육아 일기

> 3인 팀 프로젝트 | **Full-stack Developer** | Spring Boot + React + Python

아이의 성장을 기록하는 것을 넘어, AI를 통해 부모와 아이의 유대감을 시각화하는 프로젝트입니다. 전체 스택을 직접 설계하고 구현하며, 효율적인 상태 관리와 마이크로서비스 간의 매끄러운 연결을 목표로 삼았습니다.

---

## 💡 복잡한 껍데기를 벗겨내고 핵심에 집중하기

초기 개발 과정에서 `useState`가 파편화되고 Props가 계층을 타고 5단계 이상 내려가는 'Prop Drilling' 현상이 발생했습니다. 코드는 비대해졌고 데이터의 흐름을 쫓기가 점점 어려워졌습니다.

단순히 편리함을 넘어, **유지보수가 가능한 구조**를 고민했습니다. Zustand를 도입해 복잡한 상태를 한곳으로 모으고, 비즈니스 로직은 Custom Hooks로 분리해 컴포넌트는 오직 '그리는 일'에만 집중하도록 리팩토링했습니다. 그 결과, 전체 코드량을 약 40% 줄이면서도 각 기능의 역할은 더욱 선명해졌습니다.

---

## 🚀 멈추지 않는 서버, Hybrid AI 구조

AI 이미지 생성(DALL-E 3)은 결과물이 나오기까지 약 30초 이상의 시간이 걸립니다. 이를 단순하게 구현하면 메인 서버의 스레드가 묶여 다른 사용자의 요청을 처리하지 못하는 치명적인 병목이 발생했습니다.

저는 '적재적소(Right Tool for the Right Job)' 원칙을 세웠습니다.

- **Spring Boot**는 서비스의 뼈대인 가입, 권한, 트랜잭션 관리에 집중합니다.
- **Python Flask**는 얼굴 분석과 AI 연산 등 무거운 작업을 전담합니다.

이러한 분리 덕분에 AI가 이미지를 생성하는 긴 시간 동안에도, 메인 서버는 200ms 내외의 빠른 응답 속도를 유지하며 다른 사용자의 요청을 거침없이 처리할 수 있었습니다.

## 🎨 기술로 완성하는 감성: 정밀한 AI 파이프라인

"부모의 사진으로 아이 얼굴을 예측한다"는 아이디어를 현실로 만들기 위해 단순 API 호출 이상의 정밀함이 필요했습니다. 단순히 닮은 이미지를 만드는 것이 아니라, 부모의 특징이 논리적으로 반영되어야 했습니다.

1. **정밀 분석**: `face-cli`를 통해 얼굴의 특징점을 68개나 뽑아내어 눈매, 콧날, 얼굴형의 미세한 데이터를 수집했습니다.
2. **언어의 가교**: 수집된 수치 데이터를 DALL-E 3가 가장 잘 이해할 수 있는 자연어 형태의 '프롬프트'로 자동 변환하는 알고리즘을 설계했습니다.

덕분에 사용자가 사진 한 장만 올리면, 보이지 않는 곳에서 정교한 데이터 처리가 일어나 감동적인 결과물을 마주할 수 있게 되었습니다.

## 🛠️ 내가 선택한 도구들

하나의 프로젝트는 서로 다른 역할을 가진 기술들의 유기적인 결합이라고 생각합니다.

- **Backend & AI**: 안정적인 비즈니스 로직을 위한 **Java/Spring Boot**와 AI 모델 활용에 최적화된 **Python/Flask**를 조합했습니다.
- **Frontend**: **TypeScript**와 **Zustand**로 인터페이스의 안정성과 명확한 상태 관리를 챙기고, **TailwindCSS**로 빠른 UI 개발을 진행했습니다.
- **Database & Infra**: **MariaDB**에 소중한 기록을 담고, **Docker Compose**로 전체 시스템을 한 번에 실행할 수 있는 오케스트레이션 환경을 구축했습니다. **Prometheus**와 **Grafana**를 통해 시스템의 건강 상태를 실시간으로 살폈습니다.
