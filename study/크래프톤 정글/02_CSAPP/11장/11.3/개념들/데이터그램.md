# 데이터그램 (Datagram)

## 정의

데이터그램은 **네트워크 계층(IP 계층)** 에서 전송되는 **독립적인 데이터 단위**이다.  
하나의 데이터그램은 **자체적으로 목적지 주소(IP 주소)를 포함**하고 있어서,  
다른 데이터그램과 **순서나 경로에 상관없이** 네트워크를 통해 독립적으로 전송된다.

즉, 연결을 유지하거나 상태를 기억하지 않는  
**비연결형(Connectionless) 통신 단위**라고 할 수 있다.

---

## 구조

하나의 데이터그램은 다음 두 부분으로 구성된다:

1. **헤더(Header)**
    
    - 송신자 / 수신자의 IP 주소
        
    - 패킷 길이, 식별자(ID), TTL(Time To Live)
        
    - 프로토콜 식별자(TCP, UDP 등)
        
    - 오류 검출용 체크섬(Checksum)  
        이 정보들은 데이터가 목적지까지 올바르게 라우팅되도록 돕는다.
        
2. **페이로드(Payload)**
    
    - 실제 전송할 데이터
        
    - TCP, UDP 등 상위 계층의 세그먼트(segment)가 여기에 들어간다.
        

---

## 특징

| 구분    | 내용                    |
| ----- | --------------------- |
| 연결 방식 | 비연결형 (Connectionless) |
| 순서 보장 | 없음                    |
| 신뢰성   | 없음 (손실·중복·지연 가능)      |
| 전송 단위 | 독립적 패킷                |
| 경로 선택 | 각 데이터그램이 독립적으로 라우팅됨   |
| 목적    | 빠르고 단순한 네트워크 전송       |

---

## 동작 방식

1. 송신 측 네트워크 계층(IP)은 상위 계층에서 받은 데이터를 데이터그램으로 캡슐화한다.
    
2. 각 데이터그램은 라우터를 거치며, 네트워크 상태에 따라 다른 경로를 통해 전달될 수 있다.
    
3. 수신 측 IP 계층은 도착한 데이터그램의 헤더를 검사하고, 상위 계층으로 데이터를 전달한다.
    
4. 데이터그램 간 순서가 뒤바뀌거나 일부가 유실될 수 있으므로, 상위 계층(TCP 등)이 이를 복원한다.
    

---

## 장점

- **빠르고 단순한 전송 구조** — 연결 설정 과정이 없어 지연이 적다.
    
- **유연한 라우팅** — 네트워크 장애가 발생해도 다른 경로로 우회 가능.
    
- **확장성 높음** — 상태를 기억하지 않기 때문에 다수의 통신을 동시에 처리 가능.
    

---

## 단점

- **신뢰성 부족** — 손실·중복·순서 오류를 자체적으로 복구하지 못함.
    
- **오류 복구 부재** — 재전송, 확인 응답, 흐름 제어 기능이 없음.
    
- **애플리케이션 계층의 부담** — 신뢰성 확보가 필요한 경우 상위 계층(TCP 등)에서 직접 구현해야 함.
    

---

## 데이터그램 기반 프로토콜 예시

- **IP (Internet Protocol)** — 네트워크 계층에서 데이터그램 전송 담당
    
- **UDP (User Datagram Protocol)** — 전송 계층에서 데이터그램을 직접 이용
    
- **ICMP (Internet Control Message Protocol)** — 네트워크 상태·오류 보고용 데이터그램 사용
    

---

## 비교: 데이터그램 vs. 세그먼트 vs. 프레임

|구분|계층|역할|연결 방식|
|---|---|---|---|
|프레임(Frame)|링크 계층|물리 전송 단위|연결 지향 가능|
|데이터그램(Datagram)|네트워크 계층|IP 기반 독립 패킷|비연결형|
|세그먼트(Segment)|전송 계층|TCP/UDP 데이터 단위|TCP는 연결형, UDP는 비연결형|

---

## 요약

> 데이터그램은 “한 번 던지고 잊는(throw-and-forget)” 방식의 전송 단위다.  
> 빠르지만 신뢰성은 없으며,  
> **TCP/IP의 기본 운반체**로서 인터넷 전체의 데이터 이동을 담당한다.
