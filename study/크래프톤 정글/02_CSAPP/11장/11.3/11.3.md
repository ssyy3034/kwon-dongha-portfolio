# 🌐 11.3 전역 IP 인터넷 (The Global IP Internet)

## 개요

전 세계의 [[internet]]은 여러 [[LAN]]과 [[WAN]]이 라우터를 통해 연결된 **네트워크의 네트워크**다.  
이 구조는 시간이 지나며 복잡하게 발전했지만,  
**[[클라이언트-서버]]** 이라는 기본 통신 형태는 수십 년간 변하지 않았다.  
모든 호스트는 TCP/IP 프로토콜을 실행하며,  
응용 프로그램은 운영체제의 네트워크 계층을 통해 데이터를 주고받는다.

---

## TCP/IP 프로토콜

- 인터넷의 핵심 통신 구조는 여러 계층으로 구성된 프로토콜 집합이다.
    
- **[[IP]]** : 호스트 간 데이터그램(datagram) 전달을 담당한다. 신뢰성을 보장하지 않는다.
    
- **[[UDP]]** : IP 위에서 프로세스 간 비연결형 전송을 제공한다. 빠르지만 손실 가능성이 있다.
    
- **[[TCP]]** : IP 위에서 신뢰성 있고 순서가 보장되는 연결형 전송을 제공한다.
    
- 실제 네트워크 애플리케이션 대부분은 [[TCP]] 기반으로 동작한다.
    


---

## [[도메인 이름]]

- IP 주소는 기억하기 어려우므로, 사람이 읽기 쉬운 이름 형태로 대체된다.  
    예: `whaleshark.ics.cs.cmu.edu`
    
- 도메인 이름은 계층적으로 구성된다. (예: `www.twitter.com` → `twitter.com` → `.com` → root)
    
- 전 세계적으로 분산된 DNS 시스템은  
    도메인 이름과 IP 주소를 상호 변환하는 역할을 한다.
    
- 하나의 도메인이 여러 IP에 연결될 수도 있고, 여러 도메인이 하나의 IP를 공유할 수도 있다.  
	    이러한 구조는 부하 분산(로드 밸런싱)에 활용된다.
    

---

## [[인터넷 연결]]

- 한 호스트의 프로세스는 다른 호스트의 프로세스와 **연결(connection)** 을 맺어 통신할 수 있다.
    
- 연결은 [[TCP]] 또는 [[UDP]] 기반으로 이루어지며,  
    [[TCP]]는 신뢰성과 순서를 보장하고, [[UDP]]는 간단하고 빠르다.
    
- 프로그램은 소켓 인터페이스를 통해 이 연결을 생성·관리한다.
    

---

## ✅ 요약

> 인터넷은 TCP/IP 프로토콜을 기반으로 작동하며,  
> [[IP]]가 데이터의 전달 경로를 관리하고,  
> [[TCP]]가 신뢰성과 순서를 보장하며,  
> [[UDP]]가 빠르고 단순한 전송을 제공한다.  
> 사용자는 [[도메인 이름]]을 통해 접근하고,  
> 모든 통신은 [[인터넷 연결]]을 통해 이루어진다.
