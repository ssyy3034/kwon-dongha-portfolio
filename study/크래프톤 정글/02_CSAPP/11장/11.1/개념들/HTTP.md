HTTP는 웹 브라우저(클라이언트)와 웹 서버가 서로 **통신하는 방법에 대한 핵심 규칙(프로토콜)** 입니다. "이런 형식으로 요청하면, 저런 형식으로 응답한다"는 약속이죠.

---

## ## 1. HTTP의 기본 구조: 요청과 응답 (Request-Response)

HTTP 통신은 항상 클라이언트가 먼저 **요청(Request)** 을 보내고, 서버가 그에 대해 **응답(Response)** 하는 간단한 구조로 이루어집니다. 서버가 먼저 클라이언트에게 말을 거는 경우는 절대 없습니다.

- **HTTP 요청 (Request)**: 클라이언트가 서버에게 보내는 메시지입니다. "어떤 웹페이지를 보여줘" 또는 "이 데이터를 저장해 줘"와 같은 내용을 담고 있습니다.
    
- **HTTP 응답 (Response)**: 서버가 클라이언트의 요청을 처리한 후, 그 결과를 담아 보내는 메시지입니다. "요청한 웹페이지는 여기 있어" 또는 "데이터 저장이 성공적으로 완료됐어"와 같은 내용을 담습니다.
    

---

## ## 2. HTTP 메시지의 구성: 헤더 (Header)와 본문 (Body)

요청과 응답 메시지는 모두 **헤더(Header)** 와 **본문(Body)** 이라는 두 부분으로 구성됩니다.

- **헤더 (Header)**: 메시지의 **부가 정보(Metadata)** 를 담고 있습니다. 마치 소포의 송장과 같아서, 내용물이 무엇인지, 누가 보냈는지, 어떻게 처리해야 하는지 등의 중요한 정보를 담습니다.
    
    - **요청 헤더 예시**:
        
        - `Host`: 요청을 보내는 서버의 주소 (예: `Host: www.google.com`)
            
        - `User-Agent`: 클라이언트의 정보 (예: `User-Agent: Chrome/107.0 ...`)
            
        - `Accept`: 클라이언트가 받을 수 있는 데이터 타입 (예: `Accept: text/html`)
            
    - **응답 헤더 예시**:
        
        - `Content-Type`: 보내는 데이터의 종류 (예: `Content-Type: image/jpeg`)
            
        - `Content-Length`: 보내는 데이터의 크기
            
        - `Server`: 서버 소프트웨어의 정보 (예: `Server: Apache/2.4.1`)
            
- **본문 (Body)**: 메시지의 **실제 내용물**입니다. `GET` 요청처럼 단순히 정보를 조회할 때는 본문이 비어있을 수 있습니다.
    
    - **요청 본문 예시**: 로그인 시 아이디/비밀번호 정보, 게시글 작성 시 글 내용
        
    - **응답 본문 예시**: 서버가 보내주는 HTML 코드, 이미지 데이터, JSON 데이터 등

## ## 3. HTTP 메소드 (Method): 요청의 목적

메소드는 클라이언트가 서버에 **무엇을 하고 싶은지**를 나타내는 요청의 종류입니다.

- **`GET`**: 가장 흔한 메소드로, 특정 리소스(웹페이지, 이미지 등)를 **요청(GET)**할 때 사용합니다.
    
- **`POST`**: 클라이언트가 서버로 데이터를 **제출(POST)** 할 때 사용합니다. 주로 로그인, 회원가입, 게시글 작성 등 서버의 상태를 변경해야 하는 작업에 쓰입니다.
    
- **`HEAD`**: `GET`과 똑같이 요청하지만, 서버는 **실제 데이터(Body)를 제외하고 헤더 정보만** 응답으로 보내줍니다. 파일 전체를 다운로드하지 않고도 파일의 크기나 존재 여부만 빠르게 확인하고 싶을 때 유용합니다.
    
- **기타 메소드**: `PUT`(리소스 전체 수정), `DELETE`(리소스 삭제), `PATCH`(리소스 일부 수정) 등이 있습니다.

## ## 4. HTTP 상태 코드 (Status Code): 요청 처리 결과

상태 코드는 서버가 클라이언트의 요청을 처리한 **결과를 알려주는 세 자리 숫자**입니다. 이 코드를 통해 클라이언트는 자신의 요청이 성공했는지, 실패했다면 왜 실패했는지 알 수 있습니다.

- **`2xx` - 성공 (Success)**
    
    - `200 OK`: 요청이 성공적으로 처리되었습니다. 가장 일반적인 성공 코드입니다.
        
- **`3xx` - 리디렉션 (Redirection)**
    
    - `301 Moved Permanently`: 요청한 리소스의 주소가 영구적으로 변경되었습니다. 브라우저는 이 새 주소로 다시 요청을 보냅니다.
        
- **`4xx` - 클라이언트 오류 (Client Error)**
    
    - `400 Bad Request`: 요청의 형식이 잘못되어 서버가 이해할 수 없습니다.
        
    - `403 Forbidden`: 서버가 요청을 이해했지만, 접근 권한이 없어 거부했습니다.
        
    - `404 Not Found`: 요청한 리소스를 서버에서 찾을 수 없습니다. (존재하지 않는 URL)
        
- **`5xx` - 서버 오류 (Server Error)**
    
    - `500 Internal Server Error`: 서버 내부에서 요청을 처리하던 중 예상치 못한 오류가 발생했습니다.
        
    - `503 Service Unavailable`: 서버가 과부하 상태이거나 점검 중이어서 일시적으로 요청을 처리할 수 없습니다.
