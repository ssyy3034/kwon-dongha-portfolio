# 🌳 Kruskal 알고리즘 총정리

## 1) Kruskal 알고리즘이란?

- **최소 신장 트리(MST)**를 찾는 대표적인 **탐욕적(greedy) 알고리즘**
    
- 핵심 아이디어:
    
    1. 간선을 가중치 **오름차순으로 정렬**
        
    2. **사이클이 생기지 않는 선에서** 차례로 선택
        
    3. 선택된 간선으로 MST 구성

## 3) 알고리즘 절차

1. 모든 정점에 대해 **MAKE-SET** 실행 (각자 독립 집합으로 시작)
    
2. 모든 간선을 **가중치 기준 오름차순 정렬**
    
3. 정렬된 간선을 하나씩 보면서:
    
    - 두 정점이 **다른 집합**이면 → 간선 추가 & **UNION**
        
    - 두 정점이 **같은 집합**이면 → 사이클 발생 → 간선 제외
        
4. 간선이 $V-1$개 선택되면 종료
    

---

## 4) Union-Find를 이용한 사이클 검사

- **FIND(x)** → x의 집합 대표(루트) 반환
    
- **UNION(x,y)** → 두 집합을 합침
    

간선 $(u,v)$를 고려할 때:

- `FIND(u) == FIND(v)` → 같은 집합 → 사이클 발생 → 제외
    
- `FIND(u) != FIND(v)` → 다른 집합 → 안전 → 간선 추가 & UNION
    

👉 Union-Find 최적화(경로 압축 + union by rank)를 적용하면 사실상 **상수 시간**으로 사이클 검사 가능

---

## 5) 시간 복잡도

- 간선 정렬: $O(E \log E)$
    
- Union-Find로 사이클 검사: $O(E \cdot \alpha(V))$ (거의 상수 시간)
    
- 전체: **$O(E \log E)$** (정렬이 지배적)

## 6) 예시 (삼각형 그래프)

정점: A, B, C  
간선: (A,B:1), (B,C:1), (A,C:2)

1. 정렬: (A,B:1), (B,C:1), (A,C:2)
    
2. (A,B:1) 선택 → {A,B}
    
3. (B,C:1) 선택 → {A,B,C}
    
4. (A,C:2) 고려 → A, C 같은 집합 → 사이클 발생 → 제외
    

👉 MST = {(A,B), (B,C)}, 총 가중치 = **2**

## 7) 특징
