**IP 주소**는 인터넷에 연결된 각 호스트를 고유하게 식별하기 위해 할당되는 **부호 없는 32비트 정수**입니다.

네트워크 프로그램은 이 IP 주소를 `in_addr` 구조체에 저장합니다.

C

```
/* IP 주소 구조체 */
struct in_addr {
    uint32_t s_addr; /* 네트워크 바이트 순서의 주소 */
};
```

#### 네트워크 바이트 순서 (Network Byte Order)

인터넷에 연결된 호스트들은 서로 다른 바이트 순서(예: 리틀 엔디안, 빅 엔디안)를 사용할 수 있습니다. 이 문제를 해결하기 위해 TCP/IP는 IP 주소와 같이 네트워크를 통해 전달되는 모든 정수 데이터에 대해 **빅 엔디안** 방식인 **네트워크 바이트 순서**를 표준으로 정의합니다.
 
`in_addr` 구조체에 저장되는 주소는 항상 이 네트워크 바이트 순서를 따릅니다. C언어에서는 `htonl()`, `ntohl()` 같은 함수를 사용하여 호스트 바이트 순서와 네트워크 바이트 순서 간의 변환을 수행합니다.

#### 점으로 구분된 10진 표기법 (Dotted-Decimal Notation)

IP 주소는 보통 사람이 읽기 쉽도록, 4개의 바이트를 각각 10진수로 표현하고 점으로 구분하는 **점으로 구분된 10진 표기법**으로 나타냅니다.

- **예시**: `128.2.194.242`는 16진수 `0x8002c2f2`의 IP 주소를 나타냅니다.
    

`inet_pton`과 `inet_ntop` 같은 함수를 사용하여 이진 IP 주소와 10진 표기법 문자열 간의 변환을 수행할 수 있습니다.
