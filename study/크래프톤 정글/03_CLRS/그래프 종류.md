# 그래프 기본 용어 정리 (Vertex, Edge, Node, Arc)

> CLRS 20장을 기반으로 그래프(Graph)를 구성하는 가장 핵심적인 용어들의 의미와 관계를 정리합니다.

> [!TIP] 비유로 이해하기: 지하철 노선도
> 그래프를 **지하철 노선도**라고 생각하면 용어를 직관적으로 이해하기 쉽습니다.
> - **역 (Station)**: 그래프의 **점** (Vertex, Node)
> - **선로 (Track)**: 그래프의 **선** (Edge, Arc)

---

## 1. 점(Point): Vertex vs. Node

결론적으로, 알고리즘 분야에서 **Vertex와 Node는 거의 동일한 의미**로 사용됩니다.

### Vertex (정점)
- CLRS와 같은 학술적 텍스트에서 사용하는 **공식적이고 수학적인 용어**입니다.
- 그래프를 `G = (V, E)`로 정의할 때, `V`는 **정점들의 집합(Set of Vertices)**을 의미합니다.

### Node (노드)
- Vertex와 같은 대상을 가리키는, 더 **일반적이고 실용적인 용어**입니다.
- 특히 트리(Tree), 연결 리스트(Linked List) 등 다른 자료구조와의 통일성을 위해 자주 사용됩니다.

> [!NOTE] Vertex와 Node의 관계
> **Vertex가 공식 용어, Node는 흔히 쓰는 동의어**입니다. 두 용어가 나오면 같은 대상을 가리킨다고 생각해도 무방합니다.

---

## 2. 선(Connection): Edge vs. Arc

Edge와 Arc의 구분은 그래프의 **방향성(Directionality)**과 직접적인 관련이 있어 매우 중요합니다.

### Edge (간선)
- 두 정점(Vertex)을 잇는 선을 통칭하는 **가장 포괄적인 용어**입니다.
- `G = (V, E)`에서 `E`는 **간선들의 집합(Set of Edges)**을 의미합니다.
- Edge는 방향성에 따라 두 종류로 나뉩니다.
	- **무방향 간선 (Undirected Edge)**
		- 양방향 통행이 가능한 연결입니다. `{u, v}`처럼 순서 없는 집합으로 표현합니다.
	- **방향 간선 (Directed Edge)**
		- 한쪽으로만 이동할 수 있는 일방통행 연결입니다. `(u, v)`처럼 순서가 있는 쌍으로 표현합니다.

### Arc (호)
- **방향 간선(Directed Edge)만을 가리키는 구체적인 용어**입니다.
- 따라서, **방향 그래프(Directed Graph, Digraph)**에서만 사용되는 용어입니다.
- CLRS에서는 'Arc'보다 'Directed Edge'라는 표현을 주로 사용하지만, 같은 의미로 이해하면 됩니다.

> [!IMPORTANT] Edge와 Arc의 관계
> - **Edge**: 모든 종류의 선 (방향/무방향)을 포함하는 상위 개념.
> - **Arc**: 방향성이 있는 선(Directed Edge)만 지칭하는 하위 개념/동의어.

---

## 3. 수학적 정의 및 요약 (CLRS)

- 그래프 **G**는 쌍 **(V, E)**로 구성됩니다.
- **V**: **정점(Vertices)**의 유한 집합입니다.
- **E**: **간선(Edges)**의 유한 집합입니다.
	- **무방향 그래프**: E의 원소는 `{u, v}` 형태의 정점 쌍입니다.
	- **방향 그래프**: E의 원소는 `(u, v)` 형태의 정점 순서쌍이며, 이를 **호(Arc)**라고도 부릅니다.

| 구분 | 공식/주요 용어 | 동의어 또는 특수 용어 | 비유 |
| :--- | :--- | :--- | :--- |
| **점** | **Vertex (정점)** | Node (노드) | 도시, 사람, 역 |
| **선** | **Edge (간선)** | | 도로, 관계, 선로 |
| ↳ 일방향 선 | **Directed Edge** | **Arc (호)** | 일방통행 도로 |

---

## 4. 코드 표현 예시 (Python)

그래프를 코드로 표현할 때는 주로 인접 리스트(Adjacency List) 방식이 사용됩니다.

```python
# 방향 그래프(Directed Graph) 예시
# 4개의 정점(Node)과 여러 개의 간선(Arc)으로 구성

graph = {
    'A': ['B', 'C'],  # 정점 A는 B와 C로 가는 간선(Arc)을 가짐
    'B': ['D'],       # 정점 B는 D로 가는 간선(Arc)을 가짐
    'C': ['D'],       # 정점 C는 D로 가는 간선(Arc)을 가짐
    'D': []           # 정점 D는 나가는 간선이 없음
}

# graph의 key ('A', 'B', 'C', 'D')가 Vertex/Node에 해당합니다.
# value에 있는 리스트의 요소들이 Edge/Arc를 통해 연결된 도착점을 의미합니다.
```
